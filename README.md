# Fedora TV OS 2.8.0

Отдельная телевизионная Wayland-сессия для Fedora. Она работает через `labwc`, запускает приложения на весь экран и не заменяет GNOME основного пользователя.

Концепция перехода к самостоятельному загрузочному дистрибутиву, включая
архитектуру, лицензирование, драйверы, кодеки и Widevine, описана в документе
[`docs/INDEPENDENT_OS_PLAN.md`](docs/INDEPENDENT_OS_PLAN.md).

## Что уже реализовано

- отдельный пункт **Fedora TV OS** на экране входа GDM;
- отдельный непривилегированный пользователь `tv` с паролем, задаваемым прямо при установке;
- автоматическое подключение RPM Fusion Free/Nonfree, Cisco OpenH264 и системного Flathub;
- полный FFmpeg, плагины GStreamer, aptX и аппаратные VA-API-кодеки для обнаруженной графики Intel/AMD;
- полноэкранная домашняя оболочка;
- нативные приложения Fedora и Flatpak внутри отдельной TV-сессии;
- кнопки `Home`, `Super+Home` и `XF86HomePage` для возврата домой;
- смена пользователя без перезагрузки;
- веб-приложения внутри оболочки;
- современный Chromium с H.264/AAC и поддержкой защищённого видео Widevine;
- единое управление с клавиатуры, USB/Bluetooth-пульта и стандартного gamepad;
- предсказуемая навигация по сетке с памятью последнего фокуса;
- глобальные Home, громкость, мультимедийные клавиши и безопасная кнопка питания;
- детальные настройки дисплеев: выбор выхода, разрешение, частота, масштаб, поворот, позиция и Adaptive Sync;
- детальные настройки звука: громкость, отключение звука и выбор устройства вывода;
- таймер сна и настраиваемые действия кнопки питания и закрытия крышки;
- Wi‑Fi с сохранёнными профилями и автоматическим повторным подключением без ввода пароля;
- регулировка громкости, выбор сети Wi‑Fi и подключение Bluetooth-устройств прямо из интерактивной верхней панели;
- встроенный проводник для домашней папки и подключённых накопителей;
- настройки, собранные в отдельные удобные разделы вместо длинного общего списка;
- русский и английский интерфейс с экранной клавиатурой для оболочки и веб-приложений;
- компактные фирменные иконки встроенных сервисов и системные иконки установленных приложений;
- надёжный возврат из внешних системных приложений кнопкой `Home` без перекрывающих окон;
- быстрый первый экран: плитки не ждут NetworkManager, PipeWire и проверку обновлений;
- отдельная панель управления веб-приложением по кнопке Menu: назад, вперёд, воспроизведение, полноэкранный режим и переход к сайту;
- визуальный процесс обновления с системным окном подтверждения PolicyKit;
- безопасное удаление сессии без удаления пользователя и его данных.

## Установка

Сборка выполняется без прав администратора:

```bash
cd "/home/vadim/TV OS"
chmod +x scripts/*.sh session/fedora-tv-os-*
sudo dnf install rpm-build libxcrypt-compat
./scripts/build.sh
```

Затем устанавливаются RPM, необходимые репозитории и кодеки, после чего создаётся
пользователь `tv`. Установщик сам запросит пароль дважды, не показывая его в
терминале. Сам RPM регистрирует отдельную Wayland-сессию **Fedora TV OS**;
оболочка не показывается как обычное приложение:

```bash
sudo ./scripts/install-system.sh tv
```

Для автоматической установки пароль можно безопасно передать через стандартный
ввод (он не попадёт в список процессов или историю команд):

```bash
printf '%s\n' "$TV_USER_PASSWORD" | sudo ./scripts/install-system.sh tv --password-stdin
```

Если пользователь уже существует, его пароль по умолчанию сохраняется. Чтобы
заменить его интерактивно, используйте `--reset-password`. Репозитории и кодеки
можно установить или повторно проверить независимо:

```bash
sudo ./scripts/install-repositories.sh
sudo ./scripts/install-codecs.sh
```

RPM Fusion Nonfree содержит компоненты с дополнительными лицензионными и
патентными ограничениями. Перед распространением готового образа проверьте их для
целевых стран; подробности приведены в `docs/INDEPENDENT_OS_PLAN.md`.

После установки полностью завершите уже открытый сеанс пользователя `tv` и
войдите снова. Установщик назначает **Fedora TV OS** основной сессией этого
пользователя через AccountsService/GDM.

## Управление

- `↑ ↓ ← →` — перемещение фокуса без циклических прыжков между краями;
- `OK`, `Enter` или основная кнопка gamepad — открыть и подтвердить;
- `Назад`, `Escape` или `BrowserBack` — закрыть диалог, панель либо вернуться по истории сайта;
- `Home` или `XF86HomePage` — всегда вернуться на домашний экран с восстановлением последней плитки;
- `Menu`, `ContextMenu` или `F10` — открыть настройки, а внутри сайта — панель браузера;
- кнопка `⌨` — показать или скрыть экранную клавиатуру;
- `Play/Pause`, следующий, предыдущий и Stop — передать последнему веб-приложению;
- громкость и Mute — управлять системным аудиовыходом;
- кнопка питания — открыть подтверждение выключения.

Стандартные USB/Bluetooth-пульты обычно определяются как клавиатура и работают
без настройки. Gamepad использует D-pad или левый стик, `A` для выбора, `B` для
возврата и `Start` для меню. HDMI-CEC работает, если адаптер и драйвер Linux
преобразуют команды телевизора в стандартные клавиши ввода.

При первом запуске версии с поддержкой защищённого видео Fedora TV OS загрузит
компонент Widevine и один раз перезапустит оболочку для его активации. Обычные
HTML5/HLS-сервисы продолжат открываться и при недоступности компонента.

## Безопасность

- пользователь `tv` не добавляется в группу администраторов;
- пароль читается скрыто из терминала или стандартного ввода и не передаётся аргументом процесса;
- автоматический вход не включается;
- домашняя папка пользователя `vadim` недоступна пользователю `tv`;
- параметры GNOME не изменяются; выбранные действия кнопки питания и крышки хранятся в отдельном системном фрагменте `systemd-logind` и удаляются вместе с TV OS;
- системные приложения устанавливаются только после подтверждения администратора;
- удаление TV OS не удаляет домашнюю папку пользователя `tv`.

## Возврат в GNOME

Нажмите **Сменить пользователя** внизу домашнего экрана. Затем выберите пользователя `vadim` и сессию GNOME.

## Удаление TV-сессии

```bash
cd "/home/vadim/TV OS"
sudo ./scripts/uninstall-system.sh
```

Учётная запись `tv` и её файлы останутся на месте. Удалять их следует отдельно только после ручной проверки данных.
