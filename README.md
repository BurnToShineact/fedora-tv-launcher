# Fedora TV Launcher

Первая рабочая версия TV-оболочки для мини-ПК с Fedora. Приложение показывает крупные плитки веб-сервисов, открывает сайты во встроенном Chromium и управляется клавиатурой или пультом, который определяется как клавиатура.

## Дизайн версии 1.0

Интерфейс получил тему Liquid Glass: полупрозрачные панели с размытием фона, мягкие цветные блики, объёмные плитки и лёгкие анимации появления и фокуса. При включённой в системе настройке уменьшения движения анимации автоматически отключаются.

## Возможности

- полноэкранный kiosk-режим;
- управление стрелками, Enter/OK, Escape/Back и Home;
- веб-приложения в отдельном `WebContentsView`;
- добавление и запуск уже установленных системных приложений;
- сохранение cookies и авторизации в постоянном профиле;
- часы и простой интерфейс для телевизора;
- выключение, перезагрузка и управление громкостью Fedora;
- автозапуск через `.desktop` файл;
- сборка RPM или AppImage.

## Установка на Fedora

```bash
cd fedora-tv-launcher
./scripts/install-fedora.sh
npm start
```

Для запуска в обычном окне и открытия DevTools:

```bash
npm run dev
```

Выход из полноэкранного режима во время разработки: `Ctrl+Alt+H`, затем `F11`. Кнопка Home возвращает в лаунчер.

## Автозапуск

```bash
./scripts/enable-autostart.sh
```

После следующего входа в GNOME приложение запустится автоматически.

## Сборка

```bash
npm install
npm run build:rpm
# либо
npm run build:appimage
```

Результат появится в каталоге `dist/`.

## Настройка плиток

Отредактируйте `config/apps.json`:

```json
{
  "id": "example",
  "title": "Пример",
  "url": "https://example.com",
  "icon": "EX",
  "accent": "#2563eb"
}
```

Разрешены HTTPS-адреса. Плитка с `"action":"settings"` открывает локальное окно настроек.

## Кнопки пульта

- стрелки — перемещение;
- Enter/OK — открыть;
- Escape, Backspace или BrowserBack — назад;
- Home или BrowserHome — главный экран;
- аппаратные кнопки громкости — PipeWire через `wpctl`.

## Примечания

Некоторые видеосервисы используют DRM/Widevine и могут ограничивать воспроизведение во встроенном Chromium. Для них позже можно добавить внешний режим запуска Google Chrome.

Команды выключения и перезагрузки выполняются через `systemctl`. В обычной локальной сессии Fedora они обычно подтверждаются политикой Polkit; поведение зависит от настроек системы.

## Добавление собственных веб-приложений

На главном экране выберите плитку **«Добавить приложение»**. Введите название и HTTPS-адрес сайта, выберите цвет плитки и нажмите **«Добавить»**. Пользовательские приложения сохраняются в профиле Fedora TV и не пропадают после перезапуска.

Удалить пользовательскую плитку можно через **«Настройки» → «Мои приложения»**. Встроенные приложения защищены от удаления.

## Добавление установленных приложений

На главном экране выберите **«Добавить приложение» → «Из системы»**. Лаунчер покажет программы из стандартных каталогов Fedora, а также приложения Flatpak и Snap. После нажатия **«Добавить»** программа появится отдельной плиткой на главном экране.

Системные приложения запускаются по их `.desktop`-файлам через `gio`. Удаление такой плитки в настройках не удаляет саму программу с компьютера.

## Обновления через GitHub Releases

Проект настроен на автообновление AppImage с GitHub Releases. Перед первой сборкой укажите свой GitHub-аккаунт:

```bash
./scripts/configure-github.sh ИМЯ_ПОЛЬЗОВАТЕЛЯ fedora-tv-launcher
npm install
```

Создайте на GitHub публичный репозиторий с тем же именем, загрузите туда проект и отправьте тег версии:

```bash
git init
git add .
git commit -m "Fedora TV 0.2.0"
git branch -M main
git remote add origin https://github.com/ИМЯ_ПОЛЬЗОВАТЕЛЯ/fedora-tv-launcher.git
git push -u origin main
git tag v0.2.0
git push origin v0.2.0
```

GitHub Actions соберёт AppImage и создаст Release. Для следующего обновления увеличьте поле `version` в `package.json`, сделайте commit и отправьте новый тег, например `v0.2.1`.

Автообновление работает только в собранной AppImage-версии. В режиме `npm start` кнопка покажет, что приложение запущено для разработки.

## Простая установка через RPM

После создания GitHub Release скачайте файл вида:

```text
fedora-tv-launcher-0.2.0-x86_64.rpm
```

Откройте его двойным щелчком в Fedora и нажмите **«Установить»**.

Через терминал тот же пакет устанавливается так:

```bash
sudo dnf install ./fedora-tv-launcher-0.2.0-x86_64.rpm
```

Для удаления:

```bash
sudo dnf remove fedora-tv-launcher
```

RPM-версия также проверяет обновления через GitHub Releases. При установке обновления Fedora может запросить пароль администратора, поскольку системные RPM-пакеты устанавливаются через менеджер пакетов.

## RPM-сборка 1.0.0

Перед выпуском релиза убедитесь, что версия в `package.json` совпадает с тегом.
Для текущего проекта используйте тег `v1.0.0`.

GitHub Actions автоматически:

1. установит зависимости;
2. проверит совпадение версии и тега;
3. соберёт RPM;
4. прикрепит RPM к GitHub Release;
5. сохранит RPM как артефакт запуска Actions.

Установка на Fedora:

```bash
sudo dnf install ./fedora-tv-launcher-1.0.0-x86_64.rpm
```
